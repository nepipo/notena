💰 2. Was kostet was — alle Kostenblöcke im Detail

2.1 Claude API — der größte variable Kostenblock

🔄
Geändert gegenüber v1: Briefing jetzt mit Haiku statt Sonnet

Haiku 4.5 kostet $1/$5 pro MTok (input/output). Sonnet 4.6 kostet $3/$15. Für strukturierte 2–3-Satz-Briefings auf Basis bekannter Daten ist Haiku mehr als ausreichend. Ersparnis: ~83 % pro Briefing-Call.

2.2 Briefing: Cron vs. On-Demand — was das bedeutet

❌ Alt — Cron für alle Premium
100 %
aller Premium-User bekamen täglich ein Briefing — egal ob sie die App geöffnet haben oder nicht.

Bei 1.500 Premium-Usern: 1.500 Calls/Tag.

~30 %
der Premium-User öffnen die App täglich. Nur für diese wird generiert. Cache verhindert doppelte Calls am selben Tag.

Bei 1.500 Premium-Usern: ~450 Calls/Tag.

📋 6. Noch offene Schutzmaßnahmen

  • Anthropic Hard Limit ERLEDIGT
    Hard Cap gesetzt. Kein unendliches Billing mehr möglich.
  • Briefing on-demand statt Cron ERLEDIGT
    lib/actions/briefing.ts — generiert beim Dashboard-Öffnen, cached danach den ganzen Tag.
  • Haiku statt Sonnet für Briefing ERLEDIGT
    −83 % pro Briefing-Call. Qualität für 2–3-Satz-Briefings identisch.
  • Coach Rate Limit ausstehend · ~2h Aufwand
    Max. 20 Coach-Nachrichten/Stunde pro User. Blockiert Abuse ohne echte Nutzung einzuschränken. Braucht eine kleine DB-Tabelle coach_request_log.
  • Briefing nur für Premium ausstehend · Entscheidung nötig
    Aktuell bekommt jeder eingeloggte User ein Briefing (briefing_aktiv = true by default). Wenn Briefing ein Premium-Feature wird, hier ein Check auf plan_tier = 'premium' einbauen.
